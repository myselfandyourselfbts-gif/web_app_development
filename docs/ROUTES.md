# 任務管理系統 - 路由設計文件 (ROUTES)

## 1. 路由總覽表格

為了維持操作上的單一頁面流暢度，同時符合 RESTful 命名慣例，我們將主要頁面訂在首頁 `/`，其他操作皆針對任務資源 (`/tasks`) 進行處理。

| 功能 | HTTP 方法 | URL 路徑 | 對應模板 | 說明 |
| --- | --- | --- | --- | --- |
| **任務列表(首頁)** | GET | `/` | `index.html` | 顯示所有任務，兼具「新增任務」的表單 |
| **建立任務** | POST | `/tasks` | — | 接收新增表單，存入 DB，重導回 `/` |
| **切換完成狀態** | POST | `/tasks/<int:id>/toggle` | — | 切換單一任務狀態，更新 DB，重導回 `/` |
| **編輯任務頁面** | GET | `/tasks/<int:id>/edit` | `edit.html` | 顯示讓使用者修改原任務標題的表單 |
| **更新任務** | POST | `/tasks/<int:id>/update` | — | 接收編輯表單，更新 DB，重導回 `/` |
| **刪除任務** | POST | `/tasks/<int:id>/delete` | — | 從 DB 刪除單一任務，重導回 `/` |

## 2. 每個路由的詳細說明

### 任務列表 (GET `/`)
- **輸入**：無。
- **處理邏輯**：呼叫 `TaskModel.get_all()` 取得依照時間排序的任務清單。
- **輸出**：渲染 `index.html` 並傳遞任務資料。
- **錯誤處理**：若資料庫連線失敗，可回傳 500 基本錯誤頁面。

### 建立任務 (POST `/tasks`)
- **輸入**：表單欄位 `title` (文字，必填)。
- **處理邏輯**：接收後驗證字串是否為空。若有效，呼叫 `TaskModel.create(title)`。
- **輸出**：回傳 HTTP 302，重導向到首頁 `/`。
- **錯誤處理**：若標題為空，可附帶錯誤訊息重導回 `/` 或直接忽略。

### 切換完成狀態 (POST `/tasks/<int:id>/toggle`)
- **輸入**：URL 參數 `id`。
- **處理邏輯**：利用 `TaskModel.get_by_id(id)` 取得目前狀態，反轉完成狀態後呼叫 `TaskModel.update(task_id, completed=新狀態)`。
- **輸出**：重導向到首頁 `/`。
- **錯誤處理**：若找不到該任務 (404)，則直接重導回 `/`。

### 編輯任務頁面 (GET `/tasks/<int:id>/edit`)
- **輸入**：URL 參數 `id`。
- **處理邏輯**：呼叫 `TaskModel.get_by_id(id)` 取得現有任務資料。
- **輸出**：渲染 `edit.html` 並傳遞任務物件，用於填入預設值。
- **錯誤處理**：找不到該任務時回傳 404 頁面或重導回 `/`。

### 更新任務 (POST `/tasks/<int:id>/update`)
- **輸入**：URL 參數 `id`，表單欄位 `title`。
- **處理邏輯**：驗證標題不為空後，呼叫 `TaskModel.update(task_id, title=新標題)`。
- **輸出**：重導向到首頁 `/`。
- **錯誤處理**：輸入無效時重導回 `/` 或編輯頁面。

### 刪除任務 (POST `/tasks/<int:id>/delete`)
- **輸入**：URL 參數 `id`。
- **處理邏輯**：呼叫 `TaskModel.delete(task_id)`。
- **輸出**：刪除後重導向到首頁 `/`。
- **錯誤處理**：找不到對應任務則忽略並重導回 `/`。

## 3. Jinja2 模板清單

所有的模板檔案皆存放在 `app/templates/` 中。

1. **`base.html`**：
   - 全局版型（包含 `<head>`、CSS / JS 資源引入與基本網頁結構）。
   - 其他檔案皆繼承此模板。
2. **`index.html`** (繼承 `base.html`)：
   - 首頁：包含「新增任務表單輸入框」。
   - 顯示任務列表 (使用 Jinja2 `{% for task in tasks %}` 迴圈)。
   - 每個任務旁皆有獨立表單負責送出 `/toggle` 與 `/delete` POST 請求。
3. **`edit.html`** (繼承 `base.html`)：
   - 獨立的資料修改頁面。
   - 包含一個修改表單，有原本的標題作為預設值，以及一個取消按鈕導回首頁。
