# 任務管理系統 - 產品需求文件 (PRD)

## 1. 專案概述
- **背景與動機**：現代人生活與課業繁忙，常需要一個簡單明瞭的系統來追蹤各項任務的進度。建立一個直覺的任務管理系統，能夠幫助使用者減輕記憶負擔，提升工作與學習效率。
- **目標用戶**：需要管理日常工作與課業進度的個人用戶（如學生、上班族）。
- **核心價值主張**：提供一個輕量、簡單、易用的待辦事項管理工具，讓使用者能快速新增、管理及掌握這所有任務。

## 2. 功能需求
系統的主要功能包含：
1. **新增任務**
   - 作為一名個人用戶，我希望能夠新增任務，以便將未來要做的事情記錄下來。
2. **編輯任務內容**
   - 作為一名個人用戶，我希望能夠修改已建立任務的內容，以便在任務細節改變時隨時更新。
3. **標記任務完成**
   - 作為一名個人用戶，我希望能夠將已做完的任務標記為完成，以便我清楚知道哪些事情已經處理完畢。
4. **刪除不需要的任務**
   - 作為一名個人用戶，我希望能夠刪除不再需要執行的任務，以便保持待辦清單的整潔。
5. **顯示所有任務清單**
   - 作為一名個人用戶，我希望能夠在一個畫面上看到所有待辦事項，以便我一覽目前的整體進度。

## 3. 非功能需求
- **技術限制**：採用 Flask + Jinja2 + SQLite 作為基礎架構。
- **效能與安全考量**：確保操作反應迅速，基本安全防護如防範 SQL Inject 與 XSS。

## 4. MVP 範圍

### Must Have (必須具備)
- 新增任務
- 標記任務完成
- 刪除任務
- 顯示所有任務清單

### Should Have (應該具備)
- 編輯任務內容

### Nice to Have (有了更好)
- 使用者登入/註冊機制
- 任務分類或設定截止期限
